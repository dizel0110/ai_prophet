# 🆓 Принципы Разработки AI Prophet

## Главное правило: FREE-FIRST

Все решения должны использовать **бесплатные ресурсы** по умолчанию.

### Иерархия выбора технологий:

1. **Бесплатные тарифы** (Free Tier) — приоритет #1
   - Supabase (500 MB БД бесплатно)
   - Hugging Face Inference API (бесплатно с лимитами)
   - Google Gemini API (бесплатный лимит)
   - DuckDuckGo (безлимитно, без API ключа)

2. **Open-Source self-hosted** — если нет бесплатного SaaS
   - Запуск на своем оборудовании
   - Локальные модели (Ollama, LM Studio)

3. **Минимальная оплата** — только если критично необходимо
   - Предупреждать пользователя о расходах
   - Показывать альтернативы

## ✅ Текущий стек (бесплатный):

| Компонент | Решение | Стоимость |
|-----------|---------|-----------|
| LLM | Google Gemini 2.0 Flash | Free (60 запросов/мин) |
| Fallback LLM | Hugging Face Inference API | Free |
| Поиск | DuckDuckGo | Free |
| БД (локально) | SQLite | Free |
| БД (облако, план) | Supabase | Free (500 MB) |
| Хостинг | Hugging Face Spaces | Free |
| Telegram Bot | Telegram API | Free |

---

## ❌ Избегать:

- Платных API без острой необходимости
- Привязки к проприетарным сервисам с обязательной оплатой
- Решений, где нельзя оценить стоимость заранее

---

## 🔄 При обновлении зависимостей:

1. Проверять, не изменилась ли ценовая политика
2. Искать бесплатные альтернативы перед обновлением
3. Документировать все внешние зависимости в `.env.example`
