# ⚙️ AI Prophet — Обучение для администратора

## Быстрый старт (5 минут)

Хочешь сразу попробовать? Вот полный цикл:

1. **Открой Mini App** в Telegram
2. **Нажми ⚙️** (шестерёнка) → выбери **Админ**
3. Перейди на **вкладку «Админ»** → **«Клиенты»**
4. **Нажми «🧪 Создать тестового»** — появится Анна Тестова (с анкетой)
5. **Нажми на Анну** — увидишь золотую сводку, путь клиента, кнопки
6. **Нажми «🎥 Начать сеанс массажа»** → выбери 30 мин → **«Начать»**
7. Сними 5 секунд видео → **«Завершить»** → видео придёт в Telegram
8. **Нажми «🧪 Пустой (без анкеты)»** — создастся Пётр Пустой (⚠️ Нет анкеты)
9. **Нажми на Петра** → кнопка будет **«🎬 Тренировка»** (без дневника)

> **Тебе самому нужна анкета?** Напиши боту `/massage` или в Mini App: 🖐 Массаж → AI-консультация. После этого твой профиль будет с ✅ и сможешь записать полноценный сеанс на себя.

## Панель администратора
Вы управляете салоном через Mini App. Для входа не нужен пароль — система определяет вас по Telegram ID.

### Как открыть админ-панель
1. Откройте Mini App
2. Нажмите ⚙️ (шестерёнка) в правом верхнем углу
3. Переключайтесь между ролями: **Клиент → Админ → Массажист**
4. В режиме админа появляется вкладка **«Админ»**

> Режим сохраняется между сессиями.

### 🧪 Тестовые пациенты
Для демонстрации и обучения можно создать тестовых пациентов:

- **«🧪 Создать тестового»** — генерирует пациента с анкетой и историей консультаций (1-4)
- **«🧪 Пустой (без анкеты)»** — создаёт профиль без анкеты (для теста режима тренировки)

## Список пользователей и статус анкеты

В админ-панели отображаются **все пользователи**, которые хоть раз взаимодействовали с ботом или Mini App.

- **✅ Есть анкета** — у клиента заполнена анкета здоровья, доступна предсеансовая сводка
- **⚠️ Нет анкеты** — клиент не проходил консультацию, кнопка записи сеанса доступна, но предсеансовой сводки нет

Пользователи без анкеты отображаются **внизу списка**. Чтобы заполнить анкету — отправьте клиенту ссылку на бота или откройте 🖐 Массаж → «Начать консультацию».

## Управление массажистами
Администратор может назначать массажистов:
- В карте клиента кнопка **«➕ Назначить массажиста»**
- Массажист получает доступ к панели и дневнику

### Как добавить массажиста
1. В разделе **💆 Массажисты** введите Telegram ID и имя
2. Нажмите **✅ Добавить**
3. Массажист появится в списке и в бронировании

### Как редактировать имя
Нажмите **✏️** рядом с именем в списке → введите новое → Enter или клик вне поля. Esc — отмена.

> Имя массажиста отображается при бронировании (вкладка 📞 Запись → шаг выбора массажиста) и в уведомлениях. Если имя не задано — система покажет «Массажист [ID]».

## Сеанс vs Тренировка

В карте клиента кнопка записи видео различается в зависимости от статуса анкеты:

| Условие | Кнопка | Назначение |
|---------|--------|------------|
| ✅ Анкета есть | **🎥 Сеанс массажа** | Реальная работа с клиентом. Видео уходит массажисту в Telegram + запись в дневник |
| ⚠️ Анкеты нет | **🎬 Тренировка / тест** | Тестирование системы, тренировка приёмов, создание контента. Видео уходит массажисту **без** записи в дневник |

**Принцип:** без анкеты здоровья — никакой записи в дневник. Только тест системы.

## Команды Telegram для админа
| Команда | Действие |
|---------|---------|
| `/id` | Узнать свой Telegram ID |
| `/give_access @username` | Дать доступ администратору |
| `/revoke_access @username` | Забрать доступ |
| `/admin_ids` | Список всех администраторов |

> **Важно:** доступ можно давать только через `@username`, не через ID. Пользователь должен написать что-нибудь боту перед выдачей доступа.

## Команды владельца
Владелец (`OWNER_USERNAME`) может всё, что админ, плюс:
- `/vip` — управление VIP-режимом
- `/reset` — сброс пароля

## 🎬 Обучающие видео для персонала
В файлы обучения можно встраивать видео двумя способами:
- `[YOUTUBE: video_id]` — встроенный YouTube-плеер (например, `[YOUTUBE: dQw4w9WgXcQ]`)
- `[VIDEO: https://...]` — прямая ссылка на MP4-файл

Как добавить новое видео:
1. Загрузите видео на YouTube (или хостинг с прямой ссылкой на MP4)
2. Откройте `config/education_masseur.md` или `config/education_admin.md`
3. Добавьте строку `[YOUTUBE: id_вашего_видео]` в нужное место
4. Сохраните, запушите — обучение обновится автоматически

## 📅 Система записи (Booking) — для админа

Booking-система позволяет клиентам самостоятельно записываться на сеансы через Mini App или бота.

### Как это работает
1. **Клиент** → выбирает дату → время → услугу → подтверждает
2. **Массажист** → получает Telegram-уведомление
3. **Система** → хранит записи в JSON (`data/bookings.json`) и Supabase

### Управление
- Записи клиентов видны в списке клиентов админ-панели
- Статусы: `pending` (ожидает), `confirmed` (подтверждён), `cancelled` (отменён)
- Отмена возможна только в течение лимита (по умолчанию 60 мин до сеанса)
- **Автоотмена**: если массажист не подтвердил запись за 1 час до сеанса → система отменяет запись автоматически. Массажист получает напоминание за 3 часа.
- **Утренний дайджест**: в 9:00 массажистам приходит сводка всех ожидающих записей

### Технические детали
- **Модуль:** `core/booking_manager.py` — генерация слотов, CRUD, загруженность
- **Хендлер:** `handlers/booking.py` — `/book`, `/my_bookings` команды
- **Mini App:** вкладка 📞 Запись — календарь + слоты + услуги
- **API:** 7 эндпоинтов в `main.py` (`/api/massage/slots`, `/api/massage/book` и т.д.)
- **Уведомления:** `core/notifier.py` — Telegram при создании/подтверждении/отмене
- **Dual-write:** JSON fallback + Supabase (через `_sb_req` / batch POST)

### Тестирование
```bash
python -m pytest tests/test_booking.py -v  # 38 тестов
```
- Все тесты изолированы, без сети (Supabase отключён monkeypatch)
- Слоты, создание/подтверждение/отмена, workload, edge cases

### Как добавить новые услуги
В `handlers/booking.py` массив `SERVICES`:
```python
SERVICES = [
    "Классический массаж",
    "Антицеллюлитный массаж",
    # ...
]
```
В `index.html` массив `services` в функции `renderServices()` — синхронизировать вручную.

## 🗄️ Управление базой данных (Supabase)

В разделе «База данных» админ-панели есть два инструмента:

### 📤 Кнопка «Синхронизировать в БД»
Переносит все JSON-файлы из папки `data/` в Supabase.

| Когда нужна | Когда не нужна |
|------------|---------------|
| Первичное подключение Supabase к уже работающему боту | В повседневной работе — dual-write пишет сразу в оба хранилища |
| Supabase был недоступен, данные копились в JSON | При каждом добавлении массажиста / консультации |
| После рестарта HF Space (если данные не восстановились) | После перезапуска `restore_from_supabase()` восстанавливает автоматически |

### 🔄 Кнопка «Мигрировать схему БД»
Выполняет `ALTER TABLE` — добавляет недостающие колонки в таблицы.

| Когда нужна | Когда не нужна |
|------------|---------------|
| Таблицы были созданы старой версией кода | При первом запуске — `init_schema()` создаёт все колонки |
| После обновления кода без рестарта Space | При наличии `SUPABASE_DB_URL` — миграции бегут на каждом старте |

> ⚠️ Для работы кнопки требуется `SUPABASE_DB_URL` в переменных окружения (строка подключения к Postgres). Без неё кнопка покажет ошибку.
