### 2. 🧠 День: Запуск Долгосрочной Памяти (Phase 4 Start)

**Задача:** Дать Пророку первую "извилину" памяти, которая переживет перезагрузку.

#### ❓ Вопрос про базу данных в Hugging Face Space:
> **Можно ли поставить базу прямо там?**
> **Да, можно!** SQLite — это просто файл.
> Однако, **Hugging Face Spaces (бесплатные) перезагружаются** и стирают всё, что не в репозитории, если не настроить **Persistent Storage**.
>
> **Решение:**
> 1.  Мы будем использовать **SQLite** (файл `.db`).
> 2.  Для **Hugging Face** нужно будет подключить "Persistent Storage" (в настройках Space, стоит недорого или бесплатно для малых объемов) ИЛИ использовать внешнюю базу (например, бесплатный **Supabase** или **MongoDB Atlas**).
> 3.  **Для старта (завтра):** Просто файл SQLite. Локально он будет жить вечно. В Space — пока не перезагрузится контейнер (для тестов хватит).
> 4.  **Позже:** Подключим облачную базу (бесплатный тир Mongodb/Supabase), чтобы память была вечной и доступной отовсюду.

**План работ по памяти:**
- [ ] **Создать `core/memory.py`**:
    - Класс `MemoryManager`.
    - Подключение `aiosqlite`.

### 4. 🔮 Supabase MCP (перспектива)

**Задача:** Использовать официальный MCP-сервер Supabase для удобной работы с БД.

**Почему Supabase + MCP:**
- Готовый MCP-сервер от Supabase (не нужно писать свои тулы вручную).
- Автоматическая генерация инструментов для запросов к БД.
- LLM сможет напрямую выполнять SQL-запросы через MCP.
- **Бесплатно:** MCP-сервер — open-source, Supabase Free Tier — 500MB БД.

**План интеграции:**
- [ ] Создать проект на [supabase.com](https://supabase.com).
- [ ] Получить `SUPABASE_URL` и `SUPABASE_KEY`.
- [ ] Установить MCP-сервер: `npx -y @supabase/mcp-server`.
- [ ] Добавить конфигурацию MCP в проект (или запускать отдельно).
- [ ] Настроить схему БД (таблицы `user_facts`, `chat_history`, `media_cache`).
- [ ] Подключить MCP-сервер к боту — LLM получит доступ к SQL-инструментам.

**Альтернатива (если MCP не подойдет):**
- Использовать Python-клиент `supabase-py` напрямую в `core/memory.py`.
